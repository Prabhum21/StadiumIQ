# Observability and Performance Telemetry

StadiumIQ incorporates production-ready logging, real-time metrics tracking, and automated performance benchmarking.

## 1. Structured Logging

All logs generated by the system are structured JSON objects formatted on a single line. This supports simple parsing by log aggregators (e.g. Google Cloud Logging, Datadog).

### Log Schema
- `timestamp`: UTC ISO 8601 timestamp.
- `severity`: Log level (`INFO`, `WARNING`, `ERROR`).
- `component`: Source module name.
- `message`: Log text message.
- `request_id`: Tracing correlation identifier (UUID).
- `latency_ms`: Response processing latency.
- `request_path`: Target endpoint path.
- `error_details`: Traceback elements for failures.

## 2. Telemetry Metrics API (`GET /api/metrics`)

A dedicated telemetry endpoint provides real-time information regarding CPU, Memory, caching effectiveness, and Gemini usage.

## 3. Benchmarking (`scripts/bench.py`)

Developers can run simulated traffic loads to analyze response rates and latency patterns under stress conditions:

```bash
python scripts/bench.py --url http://localhost:8000/
```

This runs load test blocks of 100, 500, and 1000 requests, calculating:
- Average latency
- 95th Percentile latency (P95)
- 99th Percentile latency (P99)
- Throughput rate (requests/sec)
